# Skills Directory Index

This folder contains 11 AI-powered skills for product managers. Each skill is a standalone markdown file with trigger conditions, step-by-step instructions, self-check criteria, and output format.

## Files

| File | What It Does | When to Use |
|------|-------------|-------------|
| `prd-writer.md` | Generates problem-first PRDs with acceptance criteria and risk assessment | Starting any new feature or product spec |
| `competitive-analysis.md` | Deep competitor research with comparison matrix and strategic insights | Strategy planning, board decks, market positioning |
| `user-interview-synthesizer.md` | Extracts quotes, pain points, and themes from call transcripts | After customer calls or when synthesizing multiple interviews |
| `okr-generator.md` | Creates measurable OKRs with leading indicators and guardrails | Quarterly planning, annual goal setting |
| `launch-checklist.md` | Pre/post launch verification with compliance sections for regulated industries | Before any feature release |
| `stakeholder-brief.md` | Tailored communication adapted by audience (exec vs eng vs design vs sales) | Cross-functional updates, status reports, escalations |
| `metrics-dashboard.md` | Defines KPIs using HEART framework with full tracking plan | New feature instrumentation, dashboard design |
| `sprint-planning.md` | Sprint plans with capacity math, risk assessment, and definition of done | Sprint planning ceremonies |
| `ab-test-designer.md` | Experiment design with statistical rigor and pre-committed decision framework | Growth experiments, feature validation |
| `roadmap-builder.md` | Now/Next/Later roadmap with RICE prioritization and "Not Doing" section | Stakeholder alignment, quarterly planning |
| `task-prioritizer.md` | 4A Framework (Amplify/Accelerate/Automate/Abandon) for task prioritization | When overwhelmed with too many tasks, daily/weekly planning |

## How Skills Work
Every skill follows the same structure:
1. **Trigger** — When to activate this skill
2. **Instructions** — Step-by-step methodology (phased)
3. **Self-Check** — Validation criteria before presenting output
4. **Output Format** — Exact structure of the deliverable
