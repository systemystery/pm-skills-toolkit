# Quarterly Planning Workflow

## When to Use
2-3 weeks before a new quarter begins.

## Steps

### Step 1: Review Last Quarter
```
Pull our OKRs from last quarter. Score each KR (0.0-1.0).
What worked? What didn't? What should we carry over?
Use the OKR scoring guide from /skills/okr-generator.md.
```

### Step 2: Gather Strategy Inputs
```
Read /knowledge/company/ for updated strategy.
Check competitive landscape in /knowledge/research/.
Synthesize customer feedback from /projects/*/research/.
What are the top themes?
```

### Step 3: Generate OKR Candidates
```
Based on company strategy and last quarter's learnings,
propose 3-5 objectives with 2-4 KRs each.
Use /skills/okr-generator.md methodology.
Include leading indicators and guardrails.
```

### Step 4: Prioritize Initiatives
```
For each initiative that could drive our OKRs,
score using RICE framework from /skills/roadmap-builder.md.
Rank by score. Flag capacity constraints.
```

### Step 5: Build the Roadmap
```
Using the prioritized list, create a Now/Next/Later roadmap.
Use /skills/roadmap-builder.md.
Include a "Not Doing" section with rationale.
Map dependencies across teams.
```

### Step 6: Stakeholder Alignment
```
Create a quarterly planning deck / one-pager for leadership.
Use /skills/stakeholder-brief.md for executive audience.
Use /templates/one-pager-template.md as structure.
```

## Output
- Last quarter scorecard
- New quarter OKRs (with leading indicators + guardrails)
- Prioritized roadmap (Now/Next/Later)
- Stakeholder-ready planning doc
- Dependency map
