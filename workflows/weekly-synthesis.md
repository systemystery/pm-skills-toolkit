# Weekly Synthesis Workflow

## When to Use
End of week (Friday) or start of week (Monday morning).

## Steps

### Step 1: Gather Inputs
```
Across all projects in /projects/, summarize:
1. What shipped this week
2. What's in progress
3. What's blocked
4. Key decisions made
```

### Step 2: Metrics Check
```
Pull the latest metrics for each active project.
Flag anything that moved more than 10% in either direction.
```

### Step 3: Stakeholder Summary
```
Draft a weekly update for [my manager / my team / leadership].
Use the stakeholder-brief skill — tailor for [audience].
Include: wins, risks, asks.
```

### Step 4: Next Week Planning
```
Based on this week's progress and our OKRs, 
what should next week's priorities be?
Flag anything that needs re-prioritization.
```

### Step 5: Update Files
```
Update /tasks/current.md with next week's priorities.
Archive completed items.
Update any /knowledge/ files with new learnings.
```

## Output
- Cross-project status rollup
- Metrics dashboard with anomalies flagged
- Stakeholder-ready weekly update
- Next week's priority list
