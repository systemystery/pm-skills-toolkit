# Templates Directory Index

This folder contains ready-to-fill templates for common PM deliverables. Each template has the structure pre-built — just fill in the brackets.

## Files

| File | What It Is | When to Use |
|------|-----------|-------------|
| `prd-template.md` | Problem-first PRD structure with metrics, requirements (P0/P1/P2), risks, and timeline | Starting a new feature spec (use with `skills/prd-writer.md` for guided creation) |
| `okr-template.md` | OKR template with leading indicators, guardrails, key initiatives, and scoring guide | Quarterly planning (use with `skills/okr-generator.md` for guided creation) |
| `launch-email-template.md` | Internal launch announcement with talking points, FAQ, and team-specific actions | Announcing a feature launch to internal teams |
| `retro-template.md` | Sprint retrospective with metrics, went well/didn't go well, and action tracking | End of sprint ceremonies |
| `one-pager-template.md` | Executive one-pager with problem, opportunity, solution, metrics, and the ask | Quick stakeholder alignment, pitch decks, investment decisions |

## Templates vs Skills
- **Templates** give you the structure (fill in the blanks)
- **Skills** give you the methodology (guided creation with AI)
- Best practice: Use the skill to generate content, then format using the template
